## Постановка задачи
Необходимо разработать консольное приложение — упрощённый аналог сервиса сокращения ссылок.

### Функциональные требования:
1. **Обязательная часть:**
   - Добавление новой ссылки в систему с генерацией короткого кода.
   - Получение исходной длинной ссылки по предоставленному короткому коду.
   - Проверка существования кода в базе.
   - Вывод всех хранящихся в системе ссылок.
2. **Вариативная часть (Вариант 7):**
   - Вывод всех пар «короткий код -> длинная ссылка» в отсортированном виде по короткому коду.

## Архитектура и обоснование технологического стека

### Выбор структуры данных
В качестве основного хранилища данных был выбран встроенный связный словарь Python — `dict`. 
* **Обоснование:** Словарь в Python реализован на основе хэш-таблицы с открытой адресацией. Это обеспечивает среднюю алгоритмическую сложность операций вставки, поиска и проверки существования ключа на уровне $O(1)$. Использование обычного списка (массива) привело бы к линейному поиску за $O(n)$, что неэффективно для сервисов с большим объёмом данных.

### Выбор алгоритма генерации кода
Вместо классического хэширования исходного URL (например, через MD5 или SHA-256) в проекте использован алгоритм случайной генерации (`random.choices`) фиксированной длины (4 символа) с последующей проверкой на уникальность.
* **Обоснование:** Криптографические хэш-функции дают слишком длинные строки, что противоречит концепции *сокращателя* ссылок. Обрезка хэша до 4 символов резко повышает риск коллизий, которые невозможно разрешить из-за детерминированности хэш-функций. Случайная генерация в цикле `while` позволяет легко обойти коллизию, создав новый уникальный код.


## Тестирование и результаты работы

При запуске программа выполняет:
1. Добавление трёх тестовых URL-адресов.
2. Проверку существования одного из кодов и вывод дешифрованной ссылки.
3. Вывод всей базы в порядке добавления.
4. Вывод базы, отсортированной по алфавиту (по короткому коду).
